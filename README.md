**ProjectPilot** is the definitive "Agent Sovereign Constitution", Universal Skill Library, and Engineering Harness designed to orchestrate deterministic, professional, and context-optimized AI software development across any platform (Antigravity, Claude Code, Cursor, Windsurf, Codex, etc.).

---

## 🧠 Core Architecture (BLAST 4.0)

ProjectPilot is built on four foundational pillars, upgraded to the **BLAST 4.0 (The Hybrid Engram Harness)** standard:

### 1. Enforceable Phase DAG
All engineering workflows follow a strict, directed acyclic graph on disk (`progress/state.json`). Agents are structurally blocked from writing business logic until specifications are reviewed, and cannot deploy until quality gates are verified and signed-off.

### 2. Hybrid Memory Model
*   **Project-Local Brain (`memory_manager.js`):** A zero-dependency, ultra-fast JSON database (`.agent/memory.json`) recording session traces, micro-decisions, and blocks. It features a case-insensitive, relevance-weighted search engine.
*   **Global Semantic Brain (Pinecone):** Semantic cloud memory for saving high-level architectural precedents and patterns shared across all repositories.

### 3. Universal CLI Shell Harness
*   Colorized cross-platform CLI helper tools under `tools/shell/` (`pilot.ps1` for PowerShell and `pilot.sh` for Bash/Zsh) enable developers to query the local brain, log decisions JIT, and run quality checks directly from their standard terminal.

### 4. Agent-Agnostic Bootloader
*   **AGENTS.md:** A dense, platform-independent instruction set that binds any incoming agent to the harness discipline.
*   **spec-First Traceability:** Strictly maps EARS logic (*When, If, The system shall...*) to automated testing suites.

# 🚀 ProjectPilot: El Repositorio Maestro de Referencia (BLAST 4.0)

**ProjectPilot** es la "Constitución Soberana de Agente" definitiva, biblioteca universal de habilidades y arnés de ingeniería diseñado para orquestar un desarrollo de software profesional, determinista y optimizado por IA en cualquier plataforma (Antigravity, Claude Code, Cursor, Windsurf, Codex, etc.).

---

## 🧠 Arquitectura Central (BLAST 4.0)

ProjectPilot está estructurado sobre cuatro pilares fundamentales, ahora actualizados al estándar **BLAST 4.0 (The Hybrid Engram Harness)**:

### 1. DAG de Fases Enforzable
Todo el flujo de trabajo sigue un grafo acíclico dirigido en disco (`progress/state.json`). Los agentes tienen estrictamente prohibido programar lógica hasta tener especificaciones aprobadas, y no pueden desplegar sin superar los checkpoints de calidad.

### 2. Modelo de Memoria Híbrido
*   **Cerebro Local (`memory_manager.js`):** Base de datos JSON portátil (`.agent/memory.json`) de cero dependencias que registra trazas de sesión y micro-decisiones con un buscador rápido por concordancia de palabras clave.
*   **Cerebro Global (Pinecone):** Memoria semántica multi-idioma en la nube para guardar y recuperar patrones de diseño reutilizables.

### 3. Arnés de Consola Universal
*   Herramientas CLI multiplataforma en `tools/shell/` (`pilot.ps1` para PowerShell y `pilot.sh` para Bash/Zsh) que permiten al desarrollador e IA consultar el estado actual del DAG, guardar decisiones de arquitectura e interactuar con la memoria desde su consola nativa.

### 4. Bootloader Universal para Agentes
*   **AGENTS.md:** Un set denso de instrucciones independiente de la plataforma para sincronizar y guiar a cualquier agente de IA.
*   **Spec-First:** Mapeo obligatorio de lógica en notación EARS (*When, If, The system shall...*) con sus correspondientes pruebas unitarias automatizadas.
