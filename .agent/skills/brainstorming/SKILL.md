# Brainstorming & Spec Gathering

Your goal is to reach a "Shared Understanding" with the user before any implementation.

## When to use this skill
- When the user asks to "build X" but "X" isn't fully defined.
- To propose architectural designs or component structures.

## Workflow

1.  **Ask Discovery Questions**:
    - "What is the primary goal?"
    - "What are the core features?"
    - "What are the technical constraints?"
2.  **Propose in Chunks**: Present your understanding and design ideas in small, digestible sections.
3.  **Get Sign-off**: Explicitly ask the user if the proposed specification is correct before moving to the Planning phase.

## Instructions
- Do NOT write implementation code during this phase.
- Focus on the "What" and "Why", not the "How".
- Update `findings.md` with any new discoveries made during this session.
